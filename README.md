# Real-Estate-Price-Prediction-Statistical-Optimization
Multiple Linear Regression model in Python to predict housing prices (R2=0.662). Optimized by eliminating severe multicollinearity (VIF) and statistical noise (P-values). Features a from-scratch mathematical validation of the OLS algorithm using pure matrix algebra (NumPy).
## 📌 Project Overview
This project focuses on building a highly optimized **Multiple Linear Regression** model to predict housing prices. Rather than simply throwing data into a machine learning algorithm, this project emphasizes **Data Integrity, Statistical Rigor, and Business Interpretability**. 

The model was meticulously refined by diagnosing multicollinearity, performing hypothesis testing on features, and validating the algorithm's underlying mathematics using pure linear algebra.

## 🛠️ Tech Stack & Tools
* **Language:** Python
* **Libraries:** Pandas (Data manipulation), NumPy (Matrix algebra), Statsmodels (OLS regression & statistical testing)
* **Methodologies:** Exploratory Data Analysis (EDA), Variance Inflation Factor (VIF), Ordinary Least Squares (OLS), Feature Engineering.

## 🚀 Key Highlights & Methodology

### 1. Multicollinearity Diagnosis & Resolution
* **The Problem:** Initial models exhibited severe multicollinearity (Condition Number > 1e16), driven by perfectly correlated variables (e.g., `sqft_living` vs. `sqft_above` + `sqft_basement`).
* **The Solution:** Conducted a Variance Inflation Factor (VIF) analysis. Strategically eliminated redundant features, bringing all final VIF values strictly under **5.0** to ensure mathematical stability without sacrificing business logic.

### 2. Parsimonious Model Optimization
* Eliminated statistically insignificant noise (P-values > 0.05) through rigorous hypothesis testing.
* Maintained a robust predictive power (**$R^2$ of 0.662**) despite significantly reducing the number of variables, achieving a perfect balance between model simplicity and accuracy.
* Integrated context-rich geospatial variables (`lat`, `view`, `waterfront`) and neighborhood metrics (`sqft_living15`) to capture real-world valuation drivers.

### 3. "Under the Hood" Mathematical Validation
To validate the results generated by the `statsmodels` library, the final model's coefficients were manually calculated from scratch using NumPy and **Matrix Algebra**:
$$\beta = (X^T X)^{-1} X^T Y$$
The manually computed coefficients perfectly matched the OLS report, demonstrating a deep understanding of the underlying mathematical engine of linear regression.

## 📈 Business Impact
The resulting model is not a "black box." It provides clear, actionable insights into exactly how much each feature (e.g., an additional square foot or a waterfront view) contributes to the final property value, making it a highly interpretable tool for stakeholders and real estate analysts.
